# Script Writing Framework

Condensed from Kallaway's psychology-based scripting system. Read this before writing ANY script.

---

## The 3-Step Hook Formula

Every hook needs these 3 components in order:

### 1. Context Lean (First 1-2 sentences)
**Goal:** Topic clarity + get them leaning in

**Tactics:**
- Establish common ground
- Reference a benefit or pain point they have
- Use a metaphor to simplify a complex idea
- Tell them something that blows their mind

### 2. Scroll Stop Interjection (1 sentence)
**Goal:** Stun them. They hit a wall.

**Contrasting words:** but, however, yet, although, therefore, on the other hand

### 3. Contrarian Snapback (1-2 sentences)
**Goal:** Snap them back in the opposite direction. Bigger shock = better effect.

---

## 4 Hook Mistakes (Never Make These)

### 1. DELAY
Topic context comes too late. Fix: Topic clarity in first 1-2 seconds. Cut all fluff before it.

### 2. CONFUSION
Words/phrasing not clear enough. Fix: Fewer words, simpler words (6th grade level), active voice.

### 3. IRRELEVANCE
Viewer doesn't feel it applies to them. Fix: Use "you/your" instead of "I/me". Frame around THEIR pain point.

### 4. DISINTEREST
No curiosity loop. Fix: Create contrast between what they believe and your alternative.

---

## 3 Script Writing Mistakes

### 1. STORY FLOW
Too many tangents. Fix: Every line must advance the story. Delete distractions.

### 2. COMPREHENSION
Viewer doesn't understand. Fix: Simpler words, fewer words, active voice, restate key ideas twice.

### 3. SPEED TO VALUE
Value mention comes too late. Fix: Frontload value — give a hit in the hook or immediately after.

---

## 6 Psychology Checkpoints

Every viewer must pass through these gates. Miss one = they bounce.

### 1. PAIN ACCEPTANCE (0-3 sec)
- Go one standard deviation away from core pain (don't trigger BS detector)
- Twist the knife (remind them of frustration)
- Audience of one (use "you", talk to ONE person)
- Show the pain-solution gap

### 2. TRUST (3-10 sec)
Best to worst credibility sources:
1. Proof you solved it for someone like them
2. Proof you solved it yourself
3. Social proof (followers, views)
4. Evidence you've been in their shoes
5. Common ground
6. Professional setup

### 3. PLAN OF ATTACK (10-15 sec)
- Point of difference (contrast common belief vs your take)
- Structured list (brain loves numbers: "3 tools", "5 steps")

### 4. LIKABILITY (Throughout)
- Energy up 50%
- Simplification (make THEM feel smart)
- Authenticity (be yourself)

### 5. ATTENTION (Body)
- Create a hunt (lead with most valuable, tease more gold ahead)
- Rehooking (keep reigniting the pain-solution gap)
- Trance rhythm (vary sentence length — jagged right edge)

### 6. ACTION (CTA)
- Close the loop
- Unexpected surprise (exceed expectations)
- Quick recap of what they learned

## Value vs Process Rule

**EXPLAIN:** What the tool/product DOES for them (the value chain)
**SKIP:** How the technical implementation works (the process)

For each sentence ask: "Does this explain VALUE or PROCESS?"
- Value = keep
- Process = delete or simplify

**NEVER use the phrase "How it works is..."**

---

## The 3-Beat Rule (For Demos)

Body should have exactly 3 beats, not 12 steps:

- **Beat 1:** The setup/input
- **Beat 2:** The magic/transformation
- **Beat 3:** The result/payoff

**Wrong (12 beats):** "It triggers. Pulls data. Scrapes pages. Filters results. Downloads files. Strips text. Feeds to AI. Extracts info. Saves to database. Gets images. Generates report. Sends email."

**Right (3 beats):** "Every Sunday it scrapes the winning ads. AI pulls all the hooks, scripts, and images. Monday morning I wake up with fresh creatives ready to go."

## Script Structure

### Short-form (under 60 sec)
```
HOOK (0-3 sec) → Topic clarity + shock value
BODY (3-20 sec) → 3 beats max, clear story flow
CTA (20-25 sec) → Single clear action
```

### Short-form with rehook (30-60 sec)
```
HOOK (0-3 sec)
BODY 1 (3-15 sec) → First value delivery
REHOOK (15-20 sec) → Re-agitate, tease better coming
BODY 2 (20-40 sec) → More value
CTA (end)
```

### Rehook phrases that work:
- "Now that was good, but what's coming next is even better."
- "But here's where it gets crazy..."
- "That's cool, but watch this part."
- "And that's not even the best part."

## Writing Style Rules

| Rule | Do | Don't |
|------|-----|-------|
| Vocabulary | 6th grade level | Technical jargon |
| Voice | Active ("The tool scraped") | Passive ("The scraping was done") |
| Sentences | Short, punchy (staccato in hooks) | Long, run-on |
| Rhythm | Varied length (jagged right edge) | Same length throughout |
| Specificity | "date, vendor, amount, category" | "the data" |
| Shock value | "actually insane", specific numbers | "useful", "good" |

---

## Validation Checklist (Run Before Output)

**Hook:** Topic clear in first 2 sec? Under 20 words? Creates curiosity loop? No weak openers?

**Body:** Every sentence advances the story? No tangents? 6th grade vocabulary? 3 beats max for demos?

**CTA:** Closes the loop? Single clear action? Tied to the value shown?

**Overall:** Value not process? Emotional payoff at the end? Frontloaded the impressive part?
